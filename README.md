# My Editor ユーザーマニュアル

「My Editor」は、ブラウザ上で動作する高性能なVSCode風のWebテキストエディタです。インストールの必要がなく、社内のセキュリティ制限が厳しい環境でもブラウザから直接ローカルファイルの読み込み、編集、上書き保存、そして複数ファイルの管理が可能です。

---

## 1. アプリケーションの概要と特徴

本エディタは、Microsoftが開発しVSCodeの中核として使用されている「Monaco Editor」をベースに構築されています。そのため、VSCodeとほぼ同等の操作感と強力な開発サポート機能を備えています。

### 主な特徴
- **インストールの必要なし**: ブラウザ環境（HTTPS）があればどこでも動作します。
- **ローカルファイルへの直接上書き保存**: 編集した内容は、毎回ダウンロードすることなくPC内の元のファイルへ直接保存されます。
- **マルチ言語ハイライト**: PowerShell、JSON、JavaScript、Pythonなど、多様な言語のシンタックスハイライトに対応しています。
- **ドラッグ＆ドロップ対応**: PC上のファイルを画面内に落とすだけで、一括で複数ファイルを開くことができます。
- **快適な作業空間**: 目の負担を軽減するダークテーマと、簡単なズーム調整機能を備えています。

---

## 2. 画面構成とUI

画面は大きく分けて以下の3つのエリアで構成されています。

1. **ツールバー（上部）**
   - **「ファイルを開く」ボタン**: ローカルファイルを選択するためのボタン。
   - **タブコンテナ**: 開いているファイルのタブが横に並びます。アクティブなタブには青い上線が表示されます。
   - **ズームコントロール**: 画面の文字サイズを調整する「＋」「－」「R（リセット）」ボタン。
2. **メインエリア（中央）**
   - **エディタ画面**: ファイルが開かれている時にコードやテキストを表示・編集するエリア。
   - **空画面（案内）**: ファイルが一つも開かれていない時に表示される、ダークな背景のドラッグ＆ドロップエリア。
3. **通知領域（右下）**
   - **トースト通知**: ファイル保存時などに「保存しました！」とフワッと青いポップアップが出現します。

## 3. 基本的な使い方

### 3.1 アプリへのアクセス方法
GitHub Pagesで公開されている以下のURLにブラウザ（ChromeまたはEdgeを推奨）でアクセスします。
* `https://zumi1103.github.io/my-editor/`
* ※セキュリティ制限（File System Access APIの仕様）により、ローカルファイルを直接ダブルクリックして開いた状態（`file:///...`）では動作しません。必ず上記のHTTPS環境からアクセスしてください。

### 3.2 ファイルを開く方法
ファイルを開く方法は2通りあります。どちらの方法でも**同時に複数のファイルを選択して一括で開くことが可能**です。

#### 方法A：ボタンから開く
1. ツールバー左上の**「ファイルを開く」**ボタンをクリックします。
2. ファイル選択ダイアログが表示されます。
3. 開きたいファイルを選択し、「開く」をクリックします。
   - *複数開く場合*: `Ctrl`キーまたは`Shift`キーを押しながらファイルをクリックすることで、複数のファイルを同時に選択できます。

#### 方法B：ドラッグ＆ドロップで開く
1. PC上のフォルダやデスクトップから、開きたいファイルをマウスで掴みます（複数選択も可）。
2. ブラウザの画面（どこでも可）へ移動させ、マウスを離してドロップします。
3. 選択したすべてのファイルが一瞬でタブとして展開されます。

> **⚠️ 初回ファイル読み込み時の注意**
> ブラウザから「ファイルを編集する権限を許可しますか？」という旨の確認ポップアップ（セキュリティ警告）が表示された場合は、必ず**「許可」**または**「変更を保存する」**を選択してください。拒否すると上書き保存が機能しなくなります。

### 3.3 コードの編集
ファイルを開くと、自動的にファイルの拡張子（`.js`, `.json`, `.ps1`, `.py`など）を判別し、VSCodeと同様にキーワードや変数に色がつくシンタックスハイライトが適用されます。
また、文字を入力するとメソッドやタグの候補が自動的に表示される「入力補完（オートコンプリート）」機能も動作します。

### 3.4 タブの切り替えと閉じる方法
- **切り替え**: ツールバーに並んでいるタブをマウスクリックすると、一瞬でそのファイルのエディタ画面に切り替わります。
- **閉じる（マウス）**: タブの右側にある「×」ボタンをクリックすると、そのタブが閉じます。
- **閉じる（キーボード）**: 後述のショートカットを使用することで、マウスを操作せずに現在選択中のタブをサクサク閉じることができます。
- すべてのタブを閉じると、画面は自動的にダークな「空画面（案内）」に戻ります。

### 3.5 ファイルの上書き保存
編集が終わったら、保存の操作を行うことで、PC内の元のファイルに内容が直接上書きされます。
保存が成功すると、画面右下に青いトースト通知で**「保存しました！」**と表示され、3秒後に自動的に消えます。

### 3.6 ズーム（文字サイズ）の調整
文字が小さくて見づらい場合や、全体を俯瞰したい場合は、文字の大きさを変更できます。
- ツールバー右側の**「＋」**ボタンで拡大します。
- ツールバー右側の**「－」**ボタンで縮小します。
- ツールバー右側の**「R」**ボタンを押すと、標準の文字サイズに一発でリセットされます。

## 4. キーボードショートカット一覧

効率的に作業を行うために、以下のVSCodeに準拠した（またはブラウザ制限を回避した）便利なショートカットキーが登録されています。

| 機能 | ショートカットキー (Windows) | ショートカットキー (Mac) | 備考 |
| :--- | :--- | :--- | :--- |
| **ファイル上書き保存** | `Ctrl` + `S` | `Cmd` + `S` | ブラウザ標準の保存を無効化し、直接上書きします。成功すると右下にトーストが出ます。 |
| **アクティブなタブを閉じる** | `Alt` + `W` | `Option` + `W` | ※`Ctrl+W`はブラウザの仕様でブロックされるため、代わりにこのキーを割り当てています。連打で連続して閉じられます。 |
| **画面のズームイン（拡大）** | `Ctrl` + `マウスホイール上` <br> または `Ctrl` + `+` / `=` | `Cmd` + `マウスホイール上` <br> または `Cmd` + `+` / `=` | 目が悪い方でも快適にコードを読めるようにフォントを大きくします。 |
| **画面のズームアウト（縮小）** | `Ctrl` + `マウスホイール下` <br> または `Ctrl` + `-` | `Cmd` + `マウスホイール下` <br> または `Cmd` + `-` | コードの全体像を確認したい時に縮小します。 |
| **ズームリセット** | `Ctrl` + `0` | `Cmd` + `0` | 拡大・縮小した文字サイズを、一瞬で標準サイズに戻します。 |

## 5. サポートされているプログラミング言語の一覧

拡張子に応じて、自動的に以下の言語モードに切り替わり、適切な色付け（ハイライト）が行われます。

- **PowerShell** (`.ps1`)
- **JSON** (`.json`)
- **JavaScript** (`.js`)
- **TypeScript** (`.ts`)
- **HTML** (`.html`)
- **CSS** (`.css`)
- **Python** (`.py`)
- **XML** (`.xml`)
- **Markdown** (`.md`)
- **SQL** (`.sql`)
- **Java** (`.java`)
- **C#** (`.cs`)
- **C言語 / C++** (`.c`, `.cpp`)
- **シェルスクリプト / バッチ** (`.sh`, `.bat`)
- *上記以外*: `plaintext`（色付けなしの通常のプレーンテキスト）として開かれます。

## 6. トラブルシューティング

#### Q. 「ファイルを開く」ボタンを押しても何も起きない、または保存時にエラーが出る
- **原因1**: HTTPS環境（`https://...`）ではなく、ローカルファイル（`file:///...`）としてブラウザで直接開いていませんか？GitHub PagesのURLからアクセスし直してください。
- **原因2**: ファイルを選択した際、またはドロップした際に、ブラウザから求められた「変更の許可権限」を拒否していませんか？ページを一度リロード（`F5`）し、もう一度開き直して権限を「許可」してください。
- **原因3**: 使用しているブラウザが対応していない可能性があります。Internet Explorerや一部の古いブラウザでは動作しません。最新の **Google Chrome** または **Microsoft Edge** をご使用ください。

#### Q. 画面が真っ白になってしまった、または変更が反映されていない
- **原因**: ブラウザに古いキャッシュが残っている可能性があります。キーボードの **`Ctrl` + `F5`** （Macの場合は `Cmd` + `Shift` + `R`）を押して、強力なキャッシュクリア（スーパーリロード）を試してください。
